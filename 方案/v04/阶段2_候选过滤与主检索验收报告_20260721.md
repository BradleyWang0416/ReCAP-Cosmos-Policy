## 2. 实现与安全契约

### 2.1 来源身份与候选过滤

v04 `P2Window` 对每个 H=8/K=8 window 显式携带：

- `source_sha256`、`source_relative_path`、`source_partition`；
- `task`、`episode_id`、`role`、role content SHA；
- 当前行、8 行历史、8 行 future、projection dataset allowlist 和 pool rank。

候选过滤按 fail-closed 顺序拒绝：同 source SHA、同 source path、非 `v04_human_pool`、非 human-only、字段白名单不等于 human projection、超出 active pool、不同任务。query 必须来自 `v04_robot_dev` 或 `v04_robot_final`，并且字段白名单必须严格等于 robot-only projection。派生路径相等不再是唯一或主要安全边界。

### 2.2 主检索

主配置固定为：

| 项目 | 冻结值 |
|---|---|
| modality | `geometry_plus_visual` |
| seed | `20260711` |
| H/K | `8/8` |
| top-k | `3` |
| 主 pool | `10` |
| geometry | 历史 10D state 相对当前状态；接口要求 seen-train mean/std；L2 normalize |
| visual | 当前帧经冻结 WAN encoder 得到 latent，空间池化后 L2 normalize |
| 融合 | 两路各自归一化后按 `1/sqrt(2)` 等权拼接 |
| 同距离排序 | `SHA256(seed, query_id, human_content_sha256)` |

主配置若使用 `phase`、`oracle_phase`、非 top-k=3 或非 pool10 会 hard-fail。诊断接口只暴露 `oracle_phase`，必须提供已经完成的 primary receipt SHA256，不能在主结果前调用。

### 2.3 Feature provenance

每条 retrieval record 同时记录 query/candidate 的 source SHA、source path、partition、rank、distance、tie SHA 和两侧 feature provenance。provenance 明确列出 geometry dataset/rows、visual dataset/current row、visual feature kind，以及 future rows、target datasets、opposite-role datasets 的读取集合。

实现门禁要求：

- geometry rows 精确等于 8 行 history，最大行号不超过 current；
- visual row 精确等于 current；
- future rows、target/action dataset、opposite-role dataset 读取集合必须为空；
- 正式主检索 visual feature kind 必须为 `frozen_wan_latent`。

### 2.4 Pool growth

每任务 human pool rank 必须精确覆盖 1～10。pool1、pool2、pool4、pool8、pool10 由 rank 前缀构成，正式审计验证严格嵌套。接口同时冻结 `same_checkpoint_required=true` 与 `retraining_allowed=false`。

## 5. 非正式诊断与偏差记录

正式证据之外有三次不纳入验收计数的诊断：

1. 曾在宿主机执行一次纯静态 `py_compile`，违反“所有 Python 在 Docker 内运行”的操作约束；该命令未读取实验数据、未写正式产物、未生成 receipt，已明确标记为 `NONFORMAL_DIAGNOSTIC_DEVIATION`。后续语法、单元测试、投影审计和完整回归均在冻结 Docker 内重做，正式结论不引用该宿主机结果。
2. 第一次 Docker 聚焦测试因两个测试文件同 basename 在 pytest 收集阶段冲突，0 项测试执行、无正式产物；改为唯一文件名后重跑。
3. 第二次聚焦测试为 11 passed、1 个测试夹具的 HDF5 fancy-index 广播写法失败；失败发生在 future-target 测试数据变异语句，未进入被测检索实现。改用等价连续 slice 后 12/12 通过，并包含在最终 194 项完整套件中。

这些诊断没有改动冻结数据、模型、协议、checkpoint 或正式结果目录，不作为阶段 2 通过证据。
