## 2. 统一命令接口

唯一公开实验入口：

```bash
.venv/bin/python tools/human2robot_v04_experiment.py <subcommand> ...
```

| 子命令 | 固定参数/约束 | 阶段 3 行为 |
|---|---|---|
| `prepare-data` | v04 source/derived root | 默认 dry-run；执行仍需完整 preflight |
| `audit-data` | v04 source/derived root | 默认 dry-run；执行仍需完整 preflight |
| `prepare-features` | v04 derived/feature root | 默认 dry-run；`--execute` 在阶段 4 前 hard-block |
| `preflight` | 无 | 默认 dry-run；仅 `--execute` 运行完整四卡离线门禁 |
| `train` | `--method {no_retrieval,co_training,recap_hand_ret}` | 默认 dry-run；`--execute` 在阶段 5 前 hard-block |
| `evaluate` | `--split {dev,final}`、method、pool-size、执行时显式 checkpoint | 默认 dry-run；在阶段 6/7 前 hard-block |
| `evaluate-oracle-phase` | primary receipt SHA、pool10、执行时显式 checkpoint | oracle 只能在 primary 后诊断；当前 hard-block |
| `report` | 执行时显式 final receipt | 默认 dry-run；final 前 hard-block |

每次调用立即创建唯一 attempt，并写出 command、runtime、status、progress、日志、独立 manifest 和 receipt。manifest 与 receipt 均通过同文件系统 `.partial`、重新读取校验、原子 rename 落盘，并设置为 `0444`；同 run ID 重试只创建新的 attempt，不覆盖旧证据。

## 3. v04 独立状态机

冻结状态机恰好包含八个主状态：

```text
prepare -> preflight -> train_no_retrieval -> train_co_training
        -> train_recap_hand_ret -> dev -> final -> report
```

训练顺序固定为 no-retrieval → co-training → RECAP。oracle-phase 位于 primary 状态机之外，只能绑定已完成 primary receipt。状态机不包含 203-cell matrix，也没有 cell registry。
